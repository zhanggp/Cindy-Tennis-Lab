# Issue #002: Serve Ontology

## Why

Serve is the first concrete Skill domain. It should be modeled as a knowledge
system before becoming a training plan or Agent workflow.

## Scope

Define Serve Ontology v0.1 under `knowledge/skills/serve/`.

Initial files:

- `README.md`
- `overview.md`
- `physics.md`
- `biomechanics.md`
- `motor-learning.md`
- `common-errors.md`
- `drills.md`
- `metrics.yaml`

## Non-Scope

This issue does not include video analysis, training prescription, or automated
feedback.

## Definition Of Done

- Serve directory exists under `knowledge/skills/serve/`.
- Serve is linked to Physics, Biomechanics, Motor Learning, Drill, Metrics, and
  Experiment nodes.
- Serve-specific metrics are defined.
