# Workflow

Cindy Tennis Lab should work like an R&D project.

Each meaningful work cycle should have:

- one goal
- one output
- one commit
- one learning upgrade

## Issue Flow

1. Discuss the question or opportunity.
2. Create or choose a GitHub Issue.
3. Define the expected output.
4. Implement the smallest useful version.
5. Commit the result.
6. Record what changed in understanding.

## Suggested Sprint 1 Issues

- Issue #1: Initialize Foundation
- Issue #2: Design Knowledge Graph
- Issue #3: Serve Ontology
- Issue #4: Adult Learning System
- Issue #5: Junior Learning System

## Roles

ChatGPT helps discuss the issue and shape the thinking.

Codex helps turn the issue into files, commits, and repository history.

GitHub preserves the decisions, outputs, and evolution.
