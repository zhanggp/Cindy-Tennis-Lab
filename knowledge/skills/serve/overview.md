# Serve Overview

## Graph Position

Serve is a `Skill` node.

It depends on:

- Physics
- Biomechanics
- Anatomy
- Motor Learning
- Metrics

It produces:

- serve model
- serve drills
- serve metrics
- serve experiments
- serve-related knowledge claims

## Skill Definition

Serve is the skill of starting a point with a repeatable overhead action that
sends the ball into a target service box with an intended trajectory, spin,
placement, and recovery state.

## Core Sequence

1. Setup
2. Target intention
3. Toss
4. Trophy shape
5. Loading
6. Upward drive
7. Racket drop and acceleration
8. Contact
9. Follow-through
10. Landing
11. Recovery for the next ball

## v0.1 Model

Serve v0.1 uses four layers:

| Layer | Question |
| --- | --- |
| Physics | What must the ball do? |
| Biomechanics | What must the body coordinate? |
| Motor Learning | How does the action become repeatable? |
| Metrics | How do we know it is improving? |

## Cindy Development Rule

At age 9, the priority is action model stability, not speed.

Useful cue:

```text
抛得稳。
站得侧。
蹬得顺。
挥得松。
```

## Non-Goals

Serve Ontology v0.1 does not define:

- a daily training plan
- injury diagnosis
- video recognition
- automatic scoring
- professional-player imitation as a standard answer
