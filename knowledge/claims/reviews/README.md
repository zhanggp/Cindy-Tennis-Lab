# Claim Reviews

Claim reviews decide what happens after a claim is extracted from a source.

A claim is not automatically true. Review separates:

- claims that can become Cindy Tennis Lab operating knowledge
- claims that should remain guardrails
- claims that need experiments
- claims that should stay as research directions

## Review Decisions

| Decision | Meaning |
| --- | --- |
| `promote_to_knowledge` | Safe to promote as an operating rule, architecture rule, or guardrail |
| `needs_experiment` | Plausible, but must be tested with Mickey/Cindy data or practice review |
| `research_direction` | Useful direction, but not ready for product, training, or coaching output |
| `defer` | Not enough evidence or not relevant enough yet |

## Rules

- T0 internal decisions can be promoted when they define project operation.
- T2 official systems can become standards context or safety guardrails.
- T3 practice frameworks need validation before becoming training knowledge.
- T4 professional cases are implementation variants, not standard answers.
- T1 research can inform direction, but still needs local validation for use.
