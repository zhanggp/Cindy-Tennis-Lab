# Knowledge Capture

Turn observations into reusable knowledge.

## Inputs

- source type
- learner
- observation
- evidence reference
- principle link
- hypothesis
- next action

## Workflow

1. Restate the observation.
2. Link it to a principle or ontology area.
3. Separate known evidence from hypothesis.
4. Suggest where it belongs in `knowledge/`, `programs/`, `labs/`, or `records/`.
5. Define one follow-up review.

## Output Format

```markdown
## Knowledge Item

## Evidence

## Principle Link

## Hypothesis

## Repository Destination

## Next Review
```

## Guardrails

- Do not promote a one-time event into a universal rule.
- Preserve uncertainty.
- Prefer evidence references over memory.
